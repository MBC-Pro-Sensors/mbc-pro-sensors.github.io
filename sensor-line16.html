① 認識循行者

它是什麼？

循行者是一排 16 顆光感應器，像機器人的「腳底感官」，能偵測地面的亮暗。把它裝在機器人底部，機器人就能知道：「喔！我現在壓在黑線上了嗎？還是偏離了？」

比喻時間 🧠

想像你在矇著眼睛走路……
你只能用腳底感受「這裡是地毯？還是木頭地板？」來判斷方向。循行者就是機器人的腳底感覺——它同時有 16 根腳趾頭！每一根都在問：「這裡亮嗎？暗嗎？」

它能做什麼？

  • 循線前進：讓機器人沿著地上的黑色軌跡走
  • 判斷線的位置：知道黑線在機器人的正下方、左邊還是右邊
  • 測量線的寬度：感應器偵測到幾顆觸發，就是線有多粗
  • 高解析度定位：不只是「偏左」或「偏右」，而是精確到「偏移了多少」

② 硬體接線

就像連接樂高感應器一樣！

循行者使用標準的連接線，直接插上 SPIKE 或 EV3 主機，跟接樂高馬達完全一樣。

1

找到連接孔

感應器上的連接孔在側邊，跟樂高感應器一樣

2

拿出連接線

使用內附的標準連接線（或樂高原廠線）

3

插入主機

SPIKE 插任一 Port（A~F），EV3 插 1~4 號 Port

4

確認燈號

通電後感應器上的綠色 LED 會開始閃爍（心跳指示燈）

💡 小提醒
接線時不要用力硬插！連接頭有方向性，卡榫朝上，輕輕推進去就好。

③ 校準教學

什麼是校準？為什麼要做？

每一種地面（白紙、木板、比賽場地圖）反射的光量都不同。校準就是「讓感應器認識你的場地」，告訴它：「這個亮度是白色地面，那個亮度是黑色線。」

比喻時間 🧠
就像你換了新鞋子，要先走幾步適應腳感。校準就是讓機器人先「適應場地」，知道什麼叫亮、什麼叫暗。

校準步驟（超簡單！）

1

按下校準按鈕

按住感應器上的校準按鈕（約 2~3 秒），直到校準指示燈亮起

2

開始掃描

指示燈亮起後，將感應器在場地黑線與白色區塊上來回移動

3

保持高度

感應器距離地面 0.5 ~ 1.2 公分，不要太高也不要貼地

4

等待完成

5 秒後校準指示燈熄滅，校準完成！數值已自動儲存

⚠ 校準注意事項
  • 掃描時要同時覆蓋 最黑的黑線 和 最白的白色區域
  • 高度請維持在 0.5~1.2 公分之間，太高會讀不準
  • 校準完成後，數值會自動存入感應器記憶體，下次開機不用重做（除非換場地）

④ 各平台指令說明

不同軟體環境讀取感應器的方式不同，請依照你使用的平台選擇對應的說明。以下列出每種環境可讀取的資料內容與數值範圍。

🧱 LEGO MINDSTORMS EV3

適用：EV3 官方軟體、LEGO Classroom App、clev3r v1.7.1.0

讀取線位置 Mode 0

使用感應器積木，讀取 數值 即可取得線位置。

回傳值範圍說明
線位置-16 ~ +160 = 正中央，負數偏左，正數偏右；±16 表示線在感應器最邊緣
💡 提示
開機時若按住校準按鈕再放開，可切換追蹤黑線或白線。預設為黑線模式。

讀取 16 路反射值 Mode 1

使用進階感應器積木，讀取 數值列表（8 bytes）。

Byte內容範圍解碼方式
0ADC0 + ADC111 ~ 99十位數=ADC0，個位數=ADC1
1ADC2 + ADC311 ~ 99十位數=ADC2，個位數=ADC3
2ADC4 + ADC511 ~ 99十位數=ADC4，個位數=ADC5
3ADC6 + ADC711 ~ 99十位數=ADC6，個位數=ADC7
4ADC8 + ADC911 ~ 99十位數=ADC8，個位數=ADC9
5ADC10 + ADC1111 ~ 99十位數=ADC10，個位數=ADC11
6ADC12 + ADC1311 ~ 99十位數=ADC12，個位數=ADC13
7ADC14 + ADC1511 ~ 99十位數=ADC14，個位數=ADC15

每路 ADC 已被量化為 1~9（1=最暗，9=最亮），兩路合併為一個兩位數。

讀取位置 + 寬度 + 高解析度 Mode 2

使用進階感應器積木，讀取 數值列表（4 bytes）。

Byte內容範圍
0黑線高解析度位置待確認
1黑線寬度0 ~ 16
2白線高解析度位置待確認
3白線寬度0 ~ 16

🧱 LEGO Education SPIKE 3.4.3

適用：SPIKE App 積木模式（教育版主機 0x26）

讀取完整感應器資料 Mode 0

使用 感應器積木 → 指定模式為 模式 0，讀取數值列表（16 bytes）。

Byte內容範圍說明
0黑線寬度-128 ~ 127觸發感應器數量
1黑線位置0 ~ 31實際值 = 讀取值 - 16（範圍 -16 ~ +15）
2~3白線組合值0 ~ 3199= (白線位置+16) × 100 + 白線寬度（16-bit LE）
4ADC4~7每路 0~3bit[7:6]=ADC4, [5:4]=ADC5, [3:2]=ADC6, [1:0]=ADC7
5ADC0~3每路 0~3bit[7:6]=ADC0, [5:4]=ADC1, [3:2]=ADC2, [1:0]=ADC3
6ADC12~15每路 0~3bit[7:6]=ADC12, [5:4]=ADC13, [3:2]=ADC14, [1:0]=ADC15
7ADC8~11每路 0~3bit[7:6]=ADC8, [5:4]=ADC9, [3:2]=ADC10, [1:0]=ADC11
8~15保留0—
💡 ADC 2-bit 解碼
0 = 非常暗（黑線上），3 = 非常亮（白色地面）。每 byte 含 4 路，用位元移位取出：值 = (byte >> shift) & 0x03

🧱 LEGO MINDSTORMS Robot Inventor

適用：SPIKE 家用版 App（主機 0x25）

讀取感應器資料 Mode 0

使用 感應器積木 → 指定模式為 模式 0，讀取數值列表（8 bytes）。

Byte內容範圍說明
0黑線位置0 ~ 126127 = 0，實際值 = 讀取值 - 16（若讀取值=127 則為 0）
1黑線寬度0 ~ 10觸發感應器數量
2~3白線組合值（編碼）—特殊編碼，需解碼還原
4~5ADC0~7 二值化—8-bit 編碼，每路 > 50 為 1
6~7ADC8~15 二值化—8-bit 編碼，每路 > 50 為 1
⚠ 注意
Robot Inventor 玩具版的數值範圍與教育版不同，位置上限為 126（非 31），且 ADC 為 1-bit 二值化（僅 0/1），並非教育版的 0~3。

pyBricks MicroPython 專用，使用 hub.port.X.device.mode(9) 讀取 16 bytes 完整資料。

Byte內容範圍說明
0ADC1 + ADC0每路 0~15bit[7:4]=ADC1, [3:0]=ADC0
1ADC3 + ADC2每路 0~15bit[7:4]=ADC3, [3:0]=ADC2
2ADC5 + ADC4每路 0~15bit[7:4]=ADC5, [3:0]=ADC4
3ADC7 + ADC6每路 0~15bit[7:4]=ADC7, [3:0]=ADC6
4ADC9 + ADC8每路 0~15bit[7:4]=ADC9, [3:0]=ADC8
5ADC11 + ADC10每路 0~15bit[7:4]=ADC11, [3:0]=ADC10
6ADC13 + ADC12每路 0~15bit[7:4]=ADC13, [3:0]=ADC12
7ADC15 + ADC14每路 0~15bit[7:4]=ADC15, [3:0]=ADC14
8黑線位置0 ~ 31實際 = 讀取值 - 16
9黑線寬度0 ~ 16
10黑線高解析度0 ~ 200
11白線位置0 ~ 31實際 = 讀取值 - 16
12白線寬度0 ~ 16
13白線高解析度0 ~ 200
14~15保留0
from pybricks.hubs import SPIKEPrimeHub
from pybricks.parameters import Port

hub = SPIKEPrimeHub()

# 讀取 Mode 9 全部資料
data = hub.port.A.device.mode(9)

# 解析黑線位置（Byte 8）
blackPos = data[8] - 16 # → 範圍 -16 ~ +15

# 解析 ADC0（Byte 0 低位）
adc0 = data[0] & 0x0F # → 範圍 0 ~ 15

# 解析高解析度黑線位置（Byte 10）
blackHR = data[10] # → 範圍 0 ~ 200

⑤ 問題排解 FAQ

❓ 感應器插上去沒反應？
→ 先確認綠色 LED 有沒有在閃爍（心跳指示燈）。如果沒閃：檢查連接線是否插緊，換一個 Port 試試。如果換 Port 也沒反應：換一條連接線交叉測試。
❓ 機器人沿著線走一直晃來晃去？
→ 最常見原因：還沒校準。請回到步驟③重新校準。如果已校準但還是晃：檢查感應器高度，太近或太遠都會不準。
❓ 校準完還是讀不到線？
→ 確認校準時有掃到「最黑」和「最白」的區域。檢查場地光線是否均勻（強光直射或陰影會影響判讀）。確認感應器高度在 0.5~1.2 公分範圍內。
❓ SPIKE App 讀到的數值跟說明不一樣？
→ 確認你的主機是教育版（0x26）還是玩具版/Robot Inventor（0x25）。兩者回傳格式不同，請對照上方正確的表格。
❓ 感應器上的紅色 LED 亮了？
→ 紅色 LED 是錯誤指示燈。請先確認是否已完成校準。如果校準後仍亮紅燈，請重新校準一次。
