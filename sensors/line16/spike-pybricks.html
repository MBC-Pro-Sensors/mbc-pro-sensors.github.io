pyBricks for SPIKE

適用於在 SPIKE Prime 主機上安裝 pyBricks 韌體的使用者。支援圖控 lib.py 模式與 MicroPython Mode 9 完整資料讀取。

圖控模式：反射光 + 位置（Mode 5）

讀取 8 bytes。資料內容為 (黑線位置+16) × 100 + 黑線寬度 的編碼值。

📷 圖片位置：pyBricks Mode 5 圖控積木截圖
請將圖片命名為 pybricks-mode5-block.png 放入 images/ui/
Byte內容說明
0, 2, 4低 byte編碼值的低位元組
1, 3, 5高 byte編碼值的高位元組
6~7保留0

黑線位置範圍 -16~+16，寬度 0~16。範例程式會幫你自動解碼。

圖控模式：環境光 + 位置寬度（Mode 7）

讀取 8 bytes，包含黑線與白線的完整位置與寬度。

📷 圖片位置：pyBricks Mode 7 圖控積木截圖
請將圖片命名為 pybricks-mode7-block.png 放入 images/ui/
Byte內容範圍
0黑線位置0 ~ 32（實際 = 讀取值 - 16）
1黑線寬度0 ~ 16
2白線位置0 ~ 32（實際 = 讀取值 - 16）
3白線寬度0 ~ 16
4~5白線組合值16-bit 低位元組優先
6~7黑線組合值16-bit 低位元組優先

最強模式：一次讀取全部資料（Mode 9）

MicroPython 專用，使用 hub.port.X.device.mode(9) 讀取 16 bytes 完整資料。

📷 圖片位置：pyBricks Mode 9 程式碼截圖
請將圖片命名為 pybricks-mode9-code.png 放入 images/ui/
Byte內容範圍解碼說明
0光電反射值1 + 0每路 0~15bit[7:4]=通道1, bit[3:0]=通道0
1光電反射值3 + 2每路 0~15bit[7:4]=通道3, bit[3:0]=通道2
2光電反射值5 + 4每路 0~15bit[7:4]=通道5, bit[3:0]=通道4
3光電反射值7 + 6每路 0~15bit[7:4]=通道7, bit[3:0]=通道6
4光電反射值9 + 8每路 0~15bit[7:4]=通道9, bit[3:0]=通道8
5光電反射值11 + 10每路 0~15bit[7:4]=通道11, bit[3:0]=通道10
6光電反射值13 + 12每路 0~15bit[7:4]=通道13, bit[3:0]=通道12
7光電反射值15 + 14每路 0~15bit[7:4]=通道15, bit[3:0]=通道14
8黑線位置0 ~ 32實際值 = 讀取值 - 16（-16 ~ +16）
9黑線寬度0 ~ 16觸發感應器數量
10黑線高解析度位置0 ~ 2000=最左, 100=中央, 200=最右
11白線位置0 ~ 32實際值 = 讀取值 - 16（-16 ~ +16）
12白線寬度0 ~ 16觸發感應器數量
13白線高解析度位置0 ~ 2000=最左, 100=中央, 200=最右
14~15保留0—
from pybricks.hubs import PrimeHub
from pybricks.parameters import Port

hub = PrimeHub()

# 讀取 Mode 9 全部資料
data = hub.port.A.device.mode(9)

# 解析黑線位置（Byte 8）
blackPos = data[8] - 16 # → 範圍 -16 ~ +15

# 解析高解析度黑線位置（Byte 10）
blackHR = data[10] # → 範圍 0 ~ 200
